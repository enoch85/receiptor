## Architecture

### Authentication Flow

```
User ‚Üí Next.js App ‚Üí Kong Gateway ‚Üí Supabase Auth ‚Üí PostgreSQL
                         ‚Üì
                    JWT Token
                         ‚Üì
                  Protected Routes
```

### Session Management

- Cookies store refresh token
- Middleware auto-refreshes on each request
- Protected routes check auth status
- Redirect to login if not authenticated

### Security Features

- ‚úÖ JWT tokens (signed with secret)
- ‚úÖ HTTP-only cookies (XSS protection)
- ‚úÖ CSRF protection (Next.js built-in)
- ‚úÖ Password hashing (bcrypt via Supabase)
- ‚úÖ Email verification (can be enforced)
- ‚úÖ Rate limiting (via Kong - configurable)
